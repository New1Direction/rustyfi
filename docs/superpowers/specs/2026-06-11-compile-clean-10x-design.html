diagnosis

Context starvation, not model stupidity

Over half the residue is E0277 — and today's fixer is handed one file and a bare error string. It literally cannot see the trait the compiler says is unsatisfied, the impls that exist, or the type defined two modules away.

That's a fixable engineering problem: feed the fixer real context, and give the hardest cases an agent that can go look for itself.

03

Architecture — three components, built in measuring order

phase 1 · measure bench/ suite

~10 pinned real repos · CLI --json · scoreboard in RESULTS.md. Baseline recorded before anything changes.

→
phase 2 · enrich fix_context.rs

One-shot fixer finally sees rustc --explain, syn-extracted trait/type defs, existing impls, cross-file contracts.

→
phase 3 · close agent_fix.rs · “doctor”

Budget-capped agentic loop with tools — the only component that can do multi-file restructures.

component 1 · bench/

If we can't measure it, we can't claim it

  • bench/repos.toml — name, git url, pinned commit, language, size tier, expectation (clean | partial | impossible)
  • Selection: 5–60 source files, single dominant language, permissive license, pure-logic deps, real usage
  • CLI --json: errors, todos, files translated, duration, served model
  • bench/run.sh → bench/RESULTS.md scoreboard, re-run at every phase gate
component 2 · fix_context.rs

Give the one-shot fixer eyes

  • Per-cycle syn item index: every identifier in a diagnostic gets its full definition attached
  • rustc --explain excerpt per error code
  • E0277 special case: trait definition plus existing impl blocks for the named type
  • ~8 KB budget; definitions win over excerpts when truncating
  • Write path unchanged: single-file, snapshot-revert, src-confined
component 3 · agent_fix.rs — “rustyfi doctor”

An agent with a stethoscope, a scalpel, and a hard budget

toolsactivationbudgetssafety
list_files · read_file · search · cargo_check · explain · write_file (src-only) · done Only when errors survive cheap loop + rustfix, and only opted-in: --deep / RUSTYFI_DEEP_FIX=1. Off by default. RUSTYFI_DEEP_FIX_BUDGET: tool calls (default 40), tokens, wall clock. Cap hit → graceful exit. Snapshot src/ before phase; if final error count isn't strictly lower, revert everything. Compiler is the oracle.

Native function-calling where the provider supports it; strict JSON-action text protocol as fallback. Improves automatically as users point stronger models at it — zero code changes.

05

Risks, stated like we mean it

provider drift

Aliases move under us

DeepSeek already swapped deepseek-chat → deepseek-v4-flash mid-project. The suite pins commits and records the served model — model uplift gets acknowledged, not claimed.

cost

Doctor visits aren't free

A deep-fix session on a hard repo with a flagship model may cost a few dollars. Capped, off by default, spend reported in NEXT_STEPS.md.

impossible repos

Some apps can't be clean

Native-C-dependency projects stay in the suite marked impossible — shown in results, excluded from the clean-rate denominator. No quiet exclusions.
